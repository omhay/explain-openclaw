#### Trust Resolved Skill-Bin Paths in Allowlist Auto-Allow ([`ffd63b7a2`](https://github.com/openclaw/openclaw/commit/ffd63b7a2))

Changed `SkillBinsProvider.current()` to return `SkillBinTrustEntry` objects with resolved absolute paths, replacing the previous name-only check. New helpers `resolveExecutablePathFromEnv()` and `resolveSkillBinTrustEntries()` in `runner.ts` walk `PATH` entries to produce stable host-absolute paths. This prevents path spoofing where a relative token like `./skill-bin` could bypass the allowlist by matching a skill name without resolving to the expected binary.

#### Dedupe Shell Env Probe and Add Path Regression Test ([`204d9fb40`](https://github.com/openclaw/openclaw/commit/204d9fb40))

Extracted the login-shell environment probing loop into a dedicated `probeLoginShellEnv()` helper in `shell-env.ts`, eliminating duplicated code across probe call sites. Adds a regression test that verifies `getShellPathFromLoginShell` rejects unregistered shell paths, preventing future regressions in the shell-path validation gate.

#### Bind Node Exec Approvals to nodeId ([`4a3f8438e`](https://github.com/openclaw/openclaw/commit/4a3f8438e))

Added a `nodeId` field to exec approval requests propagated through gateway, agent, and CLI layers. `sanitizeSystemRunParamsForForwarding()` now validates that the approval's `nodeId` matches the target node before forwarding. This prevents approval reuse across different nodes — an attacker with access to one node's approval token could not previously have reused it to execute on a different node.

#### Harden Session Export Image Data-URL Handling ([`e578521ef`](https://github.com/openclaw/openclaw/commit/e578521ef))

Added `canonicalizeBase64()` to `src/media/base64.ts` to validate and normalize base64 strings before they are embedded in data URLs. The function rejects payloads containing characters that have no place in base64 (e.g., `"`, `<`, `>`), then re-encodes the decoded bytes to a clean form. Applied in `tool-images.ts`, the HTML export template, and `input-files.ts`. This closes the HTML attribute injection vector where a crafted image base64 payload could break out of a `src="data:..."` attribute context.

#### Block Twilio Webhook Replay and Stale Transitions ([`1d28da55a`](https://github.com/openclaw/openclaw/commit/1d28da55a))

Implemented per-request replay detection for Twilio and Plivo voice webhooks. A deduplication cache keyed on Twilio's idempotency token and Plivo's `X-Plivo-Signature-V2-Nonce` header blocks replayed callbacks within a 10-minute window. Speech-event resolution via `resolveTranscriptWaiter()` now requires a matching `turnToken`, preventing stale or replayed callbacks from triggering speech-to-text actions in the wrong conversation turn.

#### Centralize Sandbox Media and FS Policy Helpers ([`ce02ad964`](https://github.com/openclaw/openclaw/commit/ce02ad964))

Extracted shared sandbox media path resolution and filesystem policy helpers into new modules (`sandbox-media-paths.ts`, `tool-fs-policy.ts`), and added `unsafe-mounted-sandbox.ts` for test helper reuse. No security logic changes — purely an internal refactor that improves consistency of sandbox policy application across tools and reduces the surface area for copy-paste drift.

#### Add env -S Allowlist Bypass Regressions ([`3f923e831`](https://github.com/openclaw/openclaw/commit/3f923e831))

Adds regression tests verifying that the exec allowlist properly denies `env -S 'sh -c "payload"'` injection patterns. The `env -S` flag allows splitting a single argument into multiple arguments, making it a vector for bypassing naive name-only matching (e.g., an allowlisted `env` entry that does not account for `-S` argument parsing). These tests gate against future regressions in exec allowlist policy enforcement.

#### Harden FS Bridge Path Checks and Bind Mount Policy ([`c070be1bc`](https://github.com/openclaw/openclaw/commit/c070be1bc))

Multiple layered hardening additions to the Docker sandbox filesystem bridge and mount policy:

- Bind mount source paths are now validated against allowlist roots before being passed to the container
- Pre-existing symlinks at source paths are detected and rejected to prevent escape via host-side symlink chains
- Container-side paths are canonicalized and checked for escape (paths resolving outside the container workspace boundary are blocked)
- Reserved container targets `/workspace` and `/agent` are explicitly blocked as bind mount destinations
- New config options gate intentional overrides for these checks behind explicit opt-in flags

This directly hardens the sandbox boundary against path traversal attacks that exploit symlinks or relative path tricks in bind mount configurations.

#### Harden system.run Wrapper Enforcement ([`0026255de`](https://github.com/openclaw/openclaw/commit/0026255de))

Refactored `handleSystemRunInvoke()` with a new `SystemRunDeniedReason` type alias and `sendSystemRunDenied()` helper in `src/node-host/invoke-system-run.ts`. The refactor improves type safety for execution context parameters and denial reason propagation, ensuring all denial paths produce consistent structured responses. No behavior changes to approval logic — this is a correctness hardening that prevents silent fallthrough in denial codepaths.

#### Restore Two-Phase Approval Registration Flow ([`6f0dd6179`](https://github.com/openclaw/openclaw/commit/6f0dd6179))

Restored the two-phase approval registration pattern in `bash-tools.exec-approval-request.ts`. Phase 1 registers the approval candidate with the node; Phase 2 confirms execution before dispatch. The restored flow prevents approval bypass where single-phase approval checks could skip intermediate validation steps, allowing execution to proceed without full registration confirmation across the gateway-to-node path.

#### Harden Allow-Always Shell Multiplexer Wrapper Handling ([`a67689a7e`](https://github.com/openclaw/openclaw/commit/a67689a7e))

New module `exec-wrapper-resolution.ts` defines canonical sets of wrapper executables: `POSIX_SHELL_WRAPPERS` (ash/bash/dash/fish/ksh/sh/zsh), `DISPATCH_WRAPPER_EXECUTABLES` (sudo/env/nice/timeout/etc.), and `SHELL_MULTIPLEXER_WRAPPER_NAMES` (busybox/toybox). Sets `MAX_DISPATCH_WRAPPER_DEPTH` to 4 to prevent deep wrapper chain construction. The `exec-approvals-allowlist.ts` now uses these canonical sets when evaluating `allow-always` skill matches, preventing shell multiplexer binaries from being used as wrappers to bypass exec approval checks.

#### Require /etc/shells for Shell Env Fallback ([`ff10fe8b9`](https://github.com/openclaw/openclaw/commit/ff10fe8b9))

Tightened the shell environment fallback path in `shell-env.ts` by requiring all candidate shells to be registered in `/etc/shells`. Unregistered shell paths — including those pointing at arbitrary non-shell binaries — no longer receive login-shell environment inheritance through the `probeLoginShellEnv()` path. Adds regression tests covering the unregistered-shell rejection branch.

#### Harden autoAllowSkills Exec Matching ([`90383e00e`](https://github.com/openclaw/openclaw/commit/90383e00e))

Added `isPathScopedExecutableToken()` guard in `exec-approvals-allowlist.ts`. When `autoAllowSkills` is enabled, the skill-bin allowlist match in `evaluateSegments()` now requires a resolved path and rejects any token containing a path separator (`/` or `\`). This prevents bypass via explicit path-scoped tokens (e.g., `../../bin/sh`) that could circumvent the name-only skill matching and gain unintended auto-approval.

#### Harden Exported Session HTML Rendering ([`f8524ec77`](https://github.com/openclaw/openclaw/commit/f8524ec77))

Hardened `export-html/template.js` to prevent attribute injection in session HTML exports. The template now applies sanitization before inserting user-controlled content into innerHTML contexts. Adds 253 lines of security tests covering XSS injection vectors in session export. Part of the data-URL hardening chain initiated by `e578521ef` — both commits together address HTML injection from image payloads through to the export rendering pipeline.

#### Enforce apply_patch workspaceOnly in Sandbox Mounts ([`6634030be`](https://github.com/openclaw/openclaw/commit/6634030be))

Patched `apply-patch.ts` to propagate the `workspaceOnly` constraint through to sandbox mount resolution via `assertSandboxPath()`. Without this fix, the `apply_patch` tool could mount and access paths outside the workspace directory in sandboxed contexts, enabling path traversal to arbitrary host filesystem locations. Adds tests that verify the workspace-only constraint is enforced for sandboxed patch operations.

#### Enforce workspaceOnly for Sandbox Image Tool ([`dd9d9c1c6`](https://github.com/openclaw/openclaw/commit/dd9d9c1c6))

Added `workspaceOnly` enforcement to image path resolution in `image-tool.ts` via `assertSandboxPath()`. When the sandbox config has `workspaceOnly: true`, both the primary and fallback candidate image path resolution paths are now validated against the sandbox root. This prevents the image tool from reading (or writing) image files outside the workspace sandbox boundary, closing a path traversal vector in `nodes:image` operations.
