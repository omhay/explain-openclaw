#### Warn on likely multi-user trust-model mismatch ([`4d124e4a9`](https://github.com/openclaw/openclaw/commit/4d124e4a9))

Adds `collectLikelyMultiUserSetupFindings()` to `src/security/audit-extra.sync.ts`, called in the main security audit flow (`src/security/audit.ts`). The function detects when an OpenClaw deployment shows signs of multi-user access while using the default personal-assistant trust model. It calls `listPotentialMultiUserSignals()` to enumerate indicators — group policies with configured targets, `dmPolicy=open`, `allowFrom` wildcard patterns — and `collectRiskyToolExposureContexts()` to cross-reference with exposed tool contexts. When signals are found, the audit emits a `security.trust_model.multi_user_heuristic` finding reminding operators that OpenClaw's default trust model is personal-assistant (single trusted user), not hostile multi-tenant isolation, and that multi-user deployments require deliberate hardening.

#### Deny cron tool on /tools/invoke by default ([`d51a4695f`](https://github.com/openclaw/openclaw/commit/d51a4695f))

Adds `"cron"` to the `DEFAULT_GATEWAY_HTTP_TOOL_DENY` constant in `src/security/dangerous-tools.ts`. The cron tool is a persistent scheduling control plane — scheduling, modifying, or canceling automated runs via the HTTP `/tools/invoke` endpoint is inappropriate for a stateless, non-interactive surface. Blocking it by default prevents callers from creating persistent automated behavior through the gateway HTTP API without going through an AI session. A dedicated regression test (`tools-invoke-http.cron-regression.test.ts`) is included to guard against the deny entry being inadvertently removed.

#### Prevent tabnabbing in chat images ([`649d14152`](https://github.com/openclaw/openclaw/commit/649d14152))

Fixes a reverse tabnabbing vulnerability in the web UI chat image renderer (`ui/src/ui/chat/grouped-render.ts`). Previously, clicking an image thumbnail called `window.open(img.url, "_blank")` without `noopener`, leaving the newly opened window with a reference back to the chat page via `window.opener`. A malicious image URL target could exploit this to redirect the opener to a phishing page. The fix introduces `openImage()`, which passes `"noopener,noreferrer"` to `window.open()` and additionally sets `opened.opener = null` as a belt-and-suspenders measure. This affects only web UI users (self-hosted chat interface); terminal, desktop, and messaging platform users are unaffected. Escalated from Tier B cataloger.
