## Security Details

### HSTS Header Hardening ([`9af3ec92a`](https://github.com/openclaw/openclaw/commit/9af3ec92a))

Adds HTTP Strict-Transport-Security (HSTS) support to the gateway HTTP server. The new `setDefaultSecurityHeaders()` function in `src/gateway/http-common.ts` conditionally sets the `Strict-Transport-Security` response header alongside existing `X-Content-Type-Options` and `Referrer-Policy` headers. Configuration is controlled via `gateway.http.securityHeaders.strictTransportSecurity` in `server-runtime-config.ts`; disabled by default (no HSTS when not explicitly set, safe for non-TLS deployments). Configuration schema, type definitions, and test coverage are included.

### Telegram Reasoning-Only Leak Suppression ([`5a475259b`](https://github.com/openclaw/openclaw/commit/5a475259b))

Fixes an information disclosure where reasoning-only AI responses could be delivered via Telegram when the user has disabled reasoning output. The `splitTextIntoLaneSegments()` function in `src/telegram/bot-message-dispatch.ts` now returns a `SplitLaneSegmentsResult` with a `suppressedReasoningOnly` flag. When `resolvedReasoningLevel === "off"` and the response contains only reasoning text (no visible answer), the new `suppressedReasoningOnly` path suppresses delivery rather than sending an empty payload or leaking intermediate reasoning text.

### Cron Delivery Dispatch Refactor + Reset Notice Hardening ([`7a40d99b1`](https://github.com/openclaw/openclaw/commit/7a40d99b1))

Extracts cron agent delivery logic into a dedicated `src/cron/isolated-agent/delivery-dispatch.ts` module (NEW file). Exports `matchesMessagingToolDeliveryTarget()`, `resolveCronDeliveryBestEffort()`, and `dispatchCronDelivery()`. The "harden reset notices" aspect uses `SILENT_REPLY_TOKEN` detection in `dispatchCronDelivery()` to suppress delivery of stale parent orchestration text during active subagent runs — preventing accidental disclosure of transient intermediate state.
