### Key Security Changes

#### Rate-Limit Client Key Normalization ([`3284d2eb2`](https://github.com/openclaw/openclaw/commit/3284d2eb2))

Prevents rate-limit bypass by normalizing IPv4 and IPv4-mapped IPv6 address formats
before rate-limit key lookup. The new `normalizeRateLimitClientIp()` function in
`src/gateway/auth-rate-limit.ts` uses the existing `resolveClientIp()` from
`src/gateway/net.ts` to canonicalize addresses, ensuring that requests from `1.2.3.4`
and `::ffff:1.2.3.4` are treated as the same client for hook authentication throttling.
The internal `normalizeIp()` helper wraps `normalizeRateLimitClientIp()`, and
`resolveHookClientKey()` in `src/gateway/server-http.ts` also applies the normalization.
This closes a vector where attackers could circumvent the per-IP attempt limit by
switching between IPv4 and IPv6 representations of the same address.
