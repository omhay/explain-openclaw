### Discord Thread Binding Lifecycle Hardening ([`a7929abad`](https://github.com/openclaw/openclaw/commit/a7929abad))

Refactors Discord thread binding expiry from a single TTL to a dual-timeout model with separate `idleHours` and `maxAgeHours` semantics. The config migration function `migrateThreadBindingsTtlHoursForPath()` in `src/config/legacy.migrations.part-1.ts` migrates legacy `ttlHours` keys to `idleHours` across session, channel, and per-account scopes with validation that rejects non-finite values. Policy resolution functions `resolveThreadBindingIdleTimeoutMs()` (in `src/channels/thread-bindings-policy.ts`) and `resolveThreadBindingMaxAgeMs()` enforce timeout bounds and normalize negative values to safe defaults. The commit also re-reads live binding state before sweep operations to prevent race conditions in binding expiry.

### Canvas Capability Token Refresh Endpoint ([`54eaf1732`](https://github.com/openclaw/openclaw/commit/54eaf1732))

Adds new gateway method `node.canvas.capability.refresh` to the `NODE_ROLE_METHODS` set in `src/gateway/method-scopes.ts`, restricting it to authenticated node connections only. The handler in `src/gateway/server-methods/nodes.ts` calls `mintCanvasCapabilityToken()` from `src/gateway/canvas-capability.ts` to issue a fresh short-lived token (10-minute TTL via `CANVAS_CAPABILITY_TTL_MS`) then scopes a canvas host URL to that token via `buildCanvasScopedHostUrl()`. The method validates that a canvas host URL is present before issuing tokens and fails closed (returning UNAVAILABLE) if URL scoping fails, preventing capability tokens from being issued for unresolvable hosts.

### JSON Serialization Fix in Android Debug Handler ([`d53b24d18`](https://github.com/openclaw/openclaw/commit/d53b24d18))

Replaces manual string escaping in `apps/android/app/src/main/java/ai/openclaw/android/node/DebugHandler.kt` with Kotlin's `JsonPrimitive(diagnostics)` serializer (imported from `kotlinx.serialization.json`). The previous code used `replace("\"", "\\\"")` which did not handle newlines inside quoted JSON strings correctly, creating a vector for JSON structure corruption in diagnostic output. The fix uses proper structured serialization throughout the diagnostic handler.
