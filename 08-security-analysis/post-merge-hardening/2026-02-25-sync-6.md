#### Cap unsigned webhook body read budget ([`19d2a8998`](https://github.com/openclaw/openclaw/commit/19d2a8998))

Adds `LINE_WEBHOOK_UNSIGNED_MAX_BODY_BYTES` (4 KB) to enforce a tighter body-read budget for LINE webhooks that lack a valid `x-line-signature` header. The handler now detects missing or malformed signature headers and applies the size cap during `readBody()`, preventing denial-of-service via oversized unsigned payloads. Header array cases from Node.js are handled correctly.

#### Centralize dangerous docker override key handling ([`91ae82ae1`](https://github.com/openclaw/openclaw/commit/91ae82ae1))

Extracts the three dangerous Docker sandbox override keys into `DANGEROUS_SANDBOX_DOCKER_BOOLEAN_KEYS` and a companion helper `resolveDangerousSandboxDockerBooleans()`. Removes duplicate override-resolution logic from agent-level and global-level Docker config merging paths, making the precedence of `dangerouslyAllowReservedContainerTargets`, `dangerouslyAllowExternalBindSources`, and `dangerouslyAllowContainerNamespaceJoin` auditable in a single location.

#### Fail-closed shared-session reply routing (#24571) (thanks @brandonwise) ([`885452f5c`](https://github.com/openclaw/openclaw/commit/885452f5c))

Fixes a routing race condition in shared-session reply handling. Introduces turn-source context parameters (`turnSourceChannel`, `turnSourceTo`, `turnSourceAccountId`, `turnSourceThreadId`) that pin reply routing to the originating turn. When these parameters are present, the resolver no longer falls back to mutable session fields, preventing a concurrent inbound message from a different channel from hijacking the reply target. Implements fail-closed semantics — directly addresses the cross-channel privacy leak described in **Audit 1 Claim 6**.

#### Unify tmp alias checks and dedupe hardlink tests ([`c267b5edf`](https://github.com/openclaw/openclaw/commit/c267b5edf))

Consolidates tmp media path alias escape validation into a single `assertNoTmpAliasEscape()` helper that calls both symlink and hardlink detection in one step. Extracts `withOutsideHardlinkInOpenClawTmp()` test fixture to reduce duplication across hardlink/symlink test scenarios. Renames internal parameter from `root` to `tmpRoot` for clarity. No new security control introduced — consolidation and maintainability improvement only.

#### Share container-path utils and tighten fs bridge tests ([`eb4a93a8d`](https://github.com/openclaw/openclaw/commit/eb4a93a8d))

Extracts container path normalization into a dedicated `path-utils.ts` module exposing `normalizeContainerPath()` and `isPathInsideContainerRoot()`. Removes duplicate implementations from `fs-bridge.ts` and `fs-paths.ts`, replacing `isPathInsidePosix()` calls with the shared implementations. Tightens `fs-bridge.ts` test helpers for extracting container path arguments. Reduces the risk of a divergence between two copies of path-checking logic that could introduce a containment bypass.

#### Isolate group allowlist from pairing-store entries ([`0a5832821`](https://github.com/openclaw/openclaw/commit/0a5832821))

Removes `storeAllowList` from the `effectiveGroupAllowFrom` computation in the Nextcloud Talk inbound handler. Previously, DM-paired user entries were merged into the group allowlist check, allowing a user paired with the bot in a DM to be considered allowed for group messages. The fix ensures `effectiveGroupAllowFrom` is populated only from config-defined entries, matching the isolation applied to MSTeams (`107bda27c`) and IRC (`d1bed505c`) in the same batch.

#### Reject unsigned webhooks before body read ([`09200b3c1`](https://github.com/openclaw/openclaw/commit/09200b3c1))

Reorders the Nextcloud Talk webhook handler to verify signature headers before calling `readBody()`. Previously the request body was read before authentication was checked, allowing large unsigned payloads to consume resources. The fix uses a configurable `readBody` factory for testability and mirrors the auth-before-read pattern applied to LINE webhooks in `19d2a8998`.

#### Harden dm delivery classification ([`24d7612dd`](https://github.com/openclaw/openclaw/commit/24d7612dd))

Hardens heartbeat DM delivery classification by adding channel-specific parsers to `resolveHeartbeatDeliveryChatType()`: `resolveTelegramTargetChatType()` for Telegram and `isWhatsAppGroupJid()`/`normalizeWhatsAppTarget()` for WhatsApp. Extracts `buildNoHeartbeatDeliveryTarget()` to centralize return semantics for blocked heartbeat targets. Companion commit to `a805d6b43`, improving accuracy of the DM-blocking gate so that Telegram and WhatsApp DM targets are correctly identified and blocked.

#### Gate component command authorization for guild interactions ([`c1964e73a`](https://github.com/openclaw/openclaw/commit/c1964e73a))

Adds `resolveComponentCommandAuthorized()` to gate Discord component command interactions in guild contexts (#26119). Previously, component interactions in guilds (buttons, select menus) did not always go through the access group and allowlist checks applied to other guild commands. This commit ensures guild-scoped component interactions are authorized consistently, preventing unauthorized users from invoking interactive components.

#### Isolate group allowlist from pairing-store entries ([`107bda27c`](https://github.com/openclaw/openclaw/commit/107bda27c))

Removes stored DM-pairing entries from `effectiveGroupAllowFrom` in the MSTeams message handler, matching the isolation applied to Nextcloud Talk (`0a5832821`) and IRC (`d1bed505c`). Group allowlist checks now use only config-defined entries rather than entries from the DM pairing store, eliminating the path by which a DM-paired user could bypass group allowlist enforcement.

#### Isolate group allowlist from DM pairing store ([`d1bed505c`](https://github.com/openclaw/openclaw/commit/d1bed505c))

Applies the same group allowlist isolation as `107bda27c` (MSTeams) and `0a5832821` (Nextcloud Talk) to IRC channels. The DM pairing store is no longer merged into `effectiveGroupAllowFrom` for IRC group message authorization, ensuring group access is governed exclusively by config-defined entries.

#### Block dm targets and internalize blocked prompts ([`a805d6b43`](https://github.com/openclaw/openclaw/commit/a805d6b43))

Adds DM-target blocking to heartbeat routing via `resolveHeartbeatDeliveryChatType()`, which returns `channel: "none"` with `reason: "dm-blocked"` when the resolved delivery target is a direct message. Heartbeat messages are intended for group/channel contexts and must not be sent to DMs. Also internalizes blocked-prompt strings to remove externally observable differences in heartbeat pipeline behavior. Companion classification hardening added in `24d7612dd`.

#### Reject hardlinked tmp media aliases ([`22689b9dc`](https://github.com/openclaw/openclaw/commit/22689b9dc))

Adds `assertNoHardlinkedFinalPath()` to reject tmp media files that have more than one hard link (`stat.nlink > 1`). A file with multiple hard links can expose the same inode from outside the allowed tmp media directory, enabling path aliasing attacks where a hardlinked file inside the permitted path actually targets content outside it. This control complements existing symlink detection and is called from `assertNoTmpAliasEscape()`.
