## Defense-in-Depth Note

The `97eb5542e` commit adds a `closed` early-return guard in `src/channels/typing.ts` to prevent a race condition where a typing indicator could be sent after the reply message was delivered. This is a robustness fix for a UX bug (Telegram typing indicator lingering ~5 seconds after the bot's message appears), not a security vulnerability. The commit message uses "defense-in-depth" to describe the approach (adding an extra guard), but the issue is purely cosmetic.
