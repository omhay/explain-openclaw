#### Block @-prefixed workspace path bypass ([`9ef0fc2ff`](https://github.com/openclaw/openclaw/commit/9ef0fc2ff))

Closes a path traversal bypass where agents could prefix file paths with `@` to evade sandbox containment guards. Supplying paths like `@/etc/passwd` or `@../../sensitive` caused early normalization to pass without stripping the prefix, allowing the path to slip past `assertSandboxPath()` and `isPathInside()` checks that had no `@` awareness.

The fix applies defense-in-depth at three boundaries: `normalizeAtPrefix()` is called inside `expandPath()` to strip leading `@` at input time before any other normalization runs; `mapContainerPathToWorkspaceRoot()` normalizes `@`-prefixed container paths before remapping to the workspace root so container-aware guards are not bypassed; and `isPathInsideHost()` is refactored to validate symlink parent directories separately from the final path segment, preventing the `@` prefix from being exploited via symlink-based bypass techniques at the host-path validation layer. Test coverage verifies that `@`-prefixed absolute paths outside the workspace throw "Path escapes sandbox root" errors.

#### Block shell-wrapper positional argv approval smuggling ([`0f0a680d3`](https://github.com/openclaw/openclaw/commit/0f0a680d3))

Closes an approval bypass where an attacker could supply `bash -c "approved_command" evil_arg` to piggyback unapproved positional arguments after an inline shell `-c` command and have the approval system bind only to `approved_command` while `evil_arg` executes silently. This bypassed `validateSystemRunCommandConsistency()` because the approval display was inferred from the inline command alone.

The new `hasTrailingPositionalArgvAfterInlineCommand()` helper detects when a POSIX or PowerShell shell wrapper is combined with trailing positional args after the inline command: it calls `unwrapShellWrapperArgv()` to peel the outer shell, then checks whether `POSIX_OR_POWERSHELL_INLINE_WRAPPER_NAMES` (covering bash, sh, zsh, fish, ksh, dash, ash, pwsh, powershell) was used and whether extra positional args follow the `-c`/`-Command` flag. When trailing args are detected, `validateSystemRunCommandConsistency()` sets `mustBindDisplayToFullArgv = true`, forcing approval binding to the complete argv rather than the inferred shell command alone. Test coverage adds 19 lines verifying detection across POSIX and PowerShell shell wrapper forms.
