#### Sessions: consolidate path hardening and fallback resilience ([`6a0fcf651`](https://github.com/openclaw/openclaw/commit/6a0fcf651))

Session path resolution now hardens against symlink-based directory traversal. The new `safeRealpathSync()` helper wraps `fs.realpathSync()` with error suppression and is applied to both the sessions base directory and any candidate absolute path before performing containment checks. `resolvePathWithinSessionsDir()` canonicalizes both paths to their real filesystem locations before computing the relative path, ensuring symlinks pointing outside the sessions directory are detected even if their textual path appears within bounds. `resolvePathFromAgentSessionsDir()` applies the same canonicalization, closing the same bypass at the per-agent directory layer. The change also adds fallback resilience: `resolveSessionFilePath()` now gracefully handles stale or corrupt persisted `sessionFile` metadata that no longer resolves to a valid path.
