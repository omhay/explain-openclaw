## Summary

Thirteen security commits in this sync. Key themes:

1. **noVNC authentication hardening (`1835dec20`, `621d8e131`, `8c1518f0f`):** A three-commit chain closing unauthenticated noVNC access. First, the browser security epoch was introduced (`1835dec20`) forcing container recreation on stale security state. Then mandatory VNC password authentication was added (`621d8e131`) with the epoch bumped to `2026-02-21-novnc-auth-default`. Finally, one-time server-side tokens replaced the password-bearing URL (`8c1518f0f`) via `issueNoVncObserverToken` / `consumeNoVncObserverToken` in the new `novnc-auth.ts` module. A new audit check (`sandbox.browser_container.non_loopback_publish`, critical) detects containers publishing ports to non-loopback addresses.

2. **Sandbox browser network isolation (`f48698a50`):** Moves the sandbox browser off the shared Docker `bridge` network to a dedicated `openclaw-sandbox-browser` network. `ensureDockerNetwork` creates the dedicated network on demand and rejects dangerous modes. New `cdpSourceRange` CIDR field restricts CDP ingress; a new audit finding (`sandbox.browser_cdp_bridge_unrestricted`, warn) fires when bridge mode is used without restriction. Directly addresses Audit 2 Claim 4.

3. **Gateway auth surface hardening (`36a0df423`, `be7f82500`):** `36a0df423` introduces typed `GatewayAuthSurface` enum with explicit HTTP and WS-control-UI wrappers, eliminating the risk of wrong auth surface selection. `be7f82500` makes `X-Real-IP` header fallback opt-in (default `false`), preventing IP spoofing via unvalidated headers across all 15 gateway surfaces.

4. **Control-UI auth clarification and insecure-flag audit (`14b0d2b81`):** Clarifies that `allowInsecureAuth` does not bypass device auth. New `collectEnabledInsecureOrDangerousFlags` collector provides a consolidated audit view of all active dangerous config flags.

5. **WhatsApp security hardening (`e217f8c3f`, `10b8839a8`):** WhatsApp JID validation added across all send paths. `10b8839a8` centralizes this into `whatsapp-target-auth.ts` and fixes HTTP status codes — `ToolAuthorizationError` now returns 403 (not 400) via `resolveToolInputErrorStatus`.

6. **Heredoc shell injection protection (`f23da067f`, `92cada2ac`):** Two commits harden exec allowlist against heredoc-based shell injection (Audit 2 Claim 7). `f23da067f` requires explicit approval for heredoc commands in allowlist mode. `92cada2ac` blocks `$(`, `${`, and backtick command substitution in unquoted heredoc bodies (fixes GHSA-65rx-fvh6-r4h2, pending publication).

7. **OAuth refresh response hardening (`6cb7e16d4`):** Validates `access_token`, `expires_in` (must be positive finite number), and handles optional refresh token per RFC 6749 section 6. Addresses Audit 1 Claims 4 and 8.

8. **Telegram streaming default off (`8b1fe0d1e`):** Streaming defaults to off; silent-reply token fragments suppressed from partial previews.
