#### DNS IPv4 Prioritization in SSRF Pinned Resolver ([`d18ae2256`](https://github.com/openclaw/openclaw/commit/d18ae2256))

Introduces `dedupeAndPreferIpv4()` in `src/infra/net/ssrf.ts` to reorder DNS lookup results so IPv4 addresses appear before IPv6. This commit also unifies the channel plugin resolution and family ordering across the codebase. The IPv4-first ordering defends against misconfigured cloud hosts where IPv6 is configured but unrouted, which previously caused media fetch failures and could create inconsistent SSRF pinning behavior.

#### SSRF IPv4-First Sort Fix — Telegram Media ([`dd9ba974d`](https://github.com/openclaw/openclaw/commit/dd9ba974d))

Fixes `resolvePinnedHostnameWithPolicy()` in `src/infra/net/ssrf.ts` to sort DNS results so IPv4 addresses are always tried before IPv6. This resolved Telegram media download failures on cloud VMs (e.g., WSL2, single-stack hosts) where IPv6 was present in DNS records but unrouted. The pinned DNS path now consistently selects reachable addresses, maintaining SSRF guard integrity under round-robin DNS.

#### Sandbox Bind Parsing Centralization ([`13bfe7faa`](https://github.com/openclaw/openclaw/commit/13bfe7faa))

Extracts sandbox bind-specification parsing into a dedicated `src/agents/sandbox/bind-spec.ts` module (exporting `splitSandboxBindSpec()`) and host-path resolution into `src/agents/sandbox/host-paths.ts` (exporting `normalizeSandboxHostPath()` and `resolveSandboxHostPathViaExistingAncestor()`). The `resolveSandboxHostPathViaExistingAncestor()` function traverses the directory tree to resolve symlink parents even when the final leaf does not yet exist, blocking path traversal escapes via stale symlinks. Centralizing this logic ensures all bind-mount validation paths use the same hardened resolution.

#### Sandbox Symlink Missing-Leaf Hardening ([`b5787e4ab`](https://github.com/openclaw/openclaw/commit/b5787e4ab))

Hardens `resolveSandboxHostPathViaExistingAncestor()` in `src/agents/sandbox/host-paths.ts` to handle missing-leaf path components during bind validation. The function walks up the directory tree to find the deepest existing ancestor, resolves symlinks at that level, then reconstructs the full path with the missing segments appended. This prevents path traversal escapes where a symlink in a parent directory could redirect a non-existent bind mount target into a blocked path (e.g., `/var/run`, `/etc`). Tests cover symlink-parent escape scenarios explicitly.

#### Provider-Agnostic Talk Config Abstraction ([`d58f71571`](https://github.com/openclaw/openclaw/commit/d58f71571))

Creates `src/config/talk.ts` with `normalizeTalkConfig()` and `resolveActiveTalkProviderConfig()` to abstract Talk configuration away from ElevenLabs-specific fields. Introduces a `talk.provider` string field and `talk.providers[id]` map in the config schema, replacing hardcoded ElevenLabs key paths. The abstraction prevents accidental provider-specific API key leakage when multiple Talk providers are configured — each provider's credentials live in a namespaced sub-tree rather than at the top level where schema mismatches could expose them.

#### Timestamp Input Validation Hardening ([`2c4ebf77f`](https://github.com/openclaw/openclaw/commit/2c4ebf77f))

Adds Zod input coercion for `meta.lastTouchedAt` in `src/config/zod-schema.ts`: accepts both numeric Unix timestamps (written by older agent code using `Date.now()`) and ISO 8601 strings, coercing numbers to strings via `z.union()` with transform. Without this, numeric timestamps in saved configs caused deserialization errors that silently reset config state, a potential vector for config corruption.

#### Chat Image URL Allowlist ([`ebb568089`](https://github.com/openclaw/openclaw/commit/ebb568089))

Creates `ui/src/ui/chat/image-open.ts` with `resolveSafeImageOpenUrl()` that allowlists only `http:`, `https:`, and `blob:` protocols, plus `data:image/*` URLs with a correctly structured metadata prefix. All other data URL variants (including non-image MIME types) return null. This prevents chat-rendered images from opening arbitrary `javascript:`, `vbscript:`, or malformed data URLs via click-through — the precursor for HTML injection via unvalidated image MIME types (GHSA-2ww6-868g-2c56).

#### Safe External URL Centralization ([`e5836283a`](https://github.com/openclaw/openclaw/commit/e5836283a))

Creates `ui/src/ui/open-external-url.ts` with `resolveSafeExternalUrl()` (URL validation returning safe URL or null) and `openExternalUrlSafe()` (validated `window.open()` with `noopener,noreferrer` and explicit `opener = null`). Adds `scripts/check-no-raw-window-open.mjs` as a build-time lint script to enforce that no code calls raw `window.open()` directly. This eliminates the raw `window.open()` calls from `grouped-render.ts`, centralizing all external navigation through the safe wrapper and ensuring future UI code cannot bypass the allowlist.

#### workspaceOnly Enforcement for Image Autoload ([`370d11554`](https://github.com/openclaw/openclaw/commit/370d11554))

Reads `tools.fs.workspaceOnly` from agent config and passes it as the `workspaceOnly` option to `loadImageFromRef()` in `src/agents/pi-embedded-runner/run/images.ts`. When `workspaceOnly: true`, `assertSandboxPath()` is called to block loading images from outside the workspace directory. This fixes GHSA-q6qf-4p5j-r25g — agents could previously autoload prompt images from arbitrary filesystem paths, bypassing the `tools.fs.workspaceOnly` restriction. SECURITY.md and gateway security docs updated.

#### Session Tool Result Integrity Guard ([`8db7ca8c0`](https://github.com/openclaw/openclaw/commit/8db7ca8c0))

Fixes `src/agents/session-tool-result-guard.ts` to skip adding tool call IDs from assistant messages with `stopReason: 'aborted'` or `stopReason: 'error'` to the pending synthetic-result map. Previously, aborted tool calls were added to the pending map, causing synthetic `tool_result` blocks to be generated for incomplete calls, triggering API 400 errors (`unexpected tool_use_id found in tool_result blocks`). This aligns the write path with the existing read path in `session-transcript-repair.ts`, which already skipped aborted messages. Prevents session state corruption from orphaned tool results.

#### Exec Allowlist Bare Wildcard Fix ([`0f0b2c025`](https://github.com/openclaw/openclaw/commit/0f0b2c025))

Fixes `matchAllowlist()` in `src/infra/exec-command-resolution.ts`: the function previously skipped entries without path separators (`/`, `\`, `~`), so a bare `"*"` wildcard entry never reached the glob matcher. Even if it had, glob's single `*` maps to `[^/]*`, which would fail against absolute paths. The fix handles `"*"` as a special case that matches any resolved executable path regardless of separators. Closes #25082.

#### Exec Allowlist Wildcard Polish ([`07f653ffc`](https://github.com/openclaw/openclaw/commit/07f653ffc))

Follow-up cleanup to `src/infra/exec-command-resolution.ts` after the wildcard fix in `0f0b2c025`. Simplifies the bare wildcard handling by removing the pre-resolution guard and consolidating the pattern into a single early-return branch inside the entry loop. Net effect: same security behavior with cleaner control flow. Closes #25250.

#### Trusted-Proxy Control-UI Pairing Bypass Fix ([`20523b918`](https://github.com/openclaw/openclaw/commit/20523b918))

Adds `trustedProxyAuthOk` parameter (default `false`) to `shouldSkipControlUiPairing()` and `evaluateMissingDeviceIdentity()` in `src/gateway/server/ws-connection/connect-policy.ts`. When `gateway.auth.mode=trusted-proxy` is configured and the connection carries valid proxy headers, `trustedProxyAuthOk = true` causes both functions to allow the connection without device pairing. Fixes #25293: control-UI sessions authenticated by a trusted reverse proxy were being incorrectly rejected with "pairing required" errors despite carrying valid proxy auth.
