### a2dfe9879 — ReDoS Prevention via compileSafeRegex

Introduces a new `src/security/safe-regex.ts` module with two exported functions:
`hasNestedRepetition()` (line 17) and `compileSafeRegex()` (line 124). The module
parses regex source to detect nested quantifier patterns (e.g., `(a+)+`, `(a*)*`)
that cause exponential backtracking. Patterns with nested repetition are rejected —
`compileSafeRegex()` returns `null` instead of compiling the dangerous regex. Compiled
patterns are cached (max 256 entries with LRU eviction).

`parsePattern()` in `src/logging/redact.ts` now uses `compileSafeRegex` instead of
direct `new RegExp()` construction, protecting log redaction patterns from
operator-supplied ReDoS payloads. `matchSessionFilter()` in
`src/infra/exec-approval-forwarder.ts` similarly now uses `compileSafeRegex` for
user-supplied session filter patterns, locally closing issue #5123 (ReDoS in session
filter regex, previously WONTFIX upstream).

### 3b8e33037 — safeBins Long-Option Validation Hardened

Tightens exec-safe-bin policy to prevent long-option abbreviation bypasses.
Internal helpers added to `src/infra/exec-safe-bin-policy.ts`:
`collectKnownLongFlags()` (line 302) builds a set of canonical long option names;
`resolveCanonicalLongFlag()` (line 320) resolves abbreviated long options (e.g.,
`--randomsource` → `--random-source`) using one-match prefix resolution;
`consumeLongOptionToken()` (line 334) validates canonical form and rejects mismatched
inline values. The `sort` profile removes `--temporary-directory`, `--random-source`,
and `-T` from the allowed list as they are filesystem-dependent. Prevents GHSA-vmqr-rc7x-3446
and GHSA-4gc7-qcvf-38wg bypass vectors.

### f0c3c8b6a — Redact Dynamic Catchall Secret Keys

`src/config/redact-snapshot.ts` removes the `isExtensionPath()` guard that was
skipping regex-based redaction for unrecognized config paths. Both the redact and
restore paths now apply `isSensitivePath()` pattern matching to ALL unmatched config
paths, catching dynamic keys in catchall objects (e.g., `env.GROQ_API_KEY`) that were
not covered by schema hints. Prevents credential leakage for operator-defined keys in
dynamic config sections.

### e6484cb65 — Kilocode Auth Ordering Hardened

Enforces strict ordering: Kilocode credentials must be populated before provider
initialization. Consolidates duplicated provider wiring logic across `auth-choice-options.ts`,
`onboard-auth.config-core.ts`, `onboard-auth.credentials.ts`, and `onboard-auth.models.ts`.
Shared Kilocode constants moved to new `src/providers/kilocode-shared.ts` to prevent
circular dependencies. Ensures credentials are never undefined at provider bootstrap.

### eff3c5c70 — Session/Cron Maintenance Hardening (#24753)

Introduces session lifecycle management with disk budgeting and TTL-based pruning.
New `src/config/sessions/disk-budget.ts` adds `enforceSessionDiskBudget()`
which evicts oldest sessions when disk quota is exceeded. New `sessions cleanup` CLI
command with `--dry-run`, `--enforce`, and per-agent modes. Config fields added:
`sessions.disk.maxBytes`, `sessions.disk.highWater` (quota caps), `cron.sessionMaintenance`
(scheduling), and `sessionMaintenance.extensions` (policy extensions). Session store
gains compaction, pruning, and disk tracking. Prevents unbounded disk growth from
accumulating session transcripts.
