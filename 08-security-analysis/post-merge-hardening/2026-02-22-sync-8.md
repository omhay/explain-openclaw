### Key Security Changes

#### Open-Group Runtime/FS Exposure Detection ([`049b8b14b`](https://github.com/openclaw/openclaw/commit/049b8b14b))

Enhanced the security audit to detect risky tool exposure in open-group configurations.
`collectExposureMatrixFindings()` in `src/security/audit-extra.sync.ts` now iterates
through all agent contexts, calling `resolveSandboxConfigForAgent()` and
`isToolAllowedByPolicies()` to identify when runtime tools (`exec`, `process`) or
filesystem tools (`read`, `write`, `edit`, `apply_patch`) are enabled without full
sandbox mode protection. Raises critical severity when `groupPolicy="open"` coexists
with unguarded runtime tool access — directly addressing prompt-injection attack vectors.

#### Avatar Validation and Size Limits ([`e0db04a50`](https://github.com/openclaw/openclaw/commit/e0db04a50))

Centralizes avatar validation into a new `src/shared/avatar-policy.ts` module.
Introduces `AVATAR_MAX_BYTES` (2 MB limit), `AVATAR_DATA_RE`, `AVATAR_IMAGE_DATA_RE`,
`AVATAR_HTTP_RE`, and `AVATAR_SCHEME_RE` for scheme validation, plus
`resolveAvatarMime()` for safe MIME resolution. Consolidates previously scattered
validation across `identity-avatar.ts`, `validation.ts`, `assistant-identity.ts`,
`control-ui-shared.ts`, and `session-utils.ts`.

#### Shell-Wrapper Approval Persistence ([`98b2b16ac`](https://github.com/openclaw/openclaw/commit/98b2b16ac))

Adds inner command persistence for shell-wrapper approvals. New functions
`normalizeSafeBins()` and `resolveSafeBins()` in `src/infra/exec-approvals-allowlist.ts`
normalize and resolve safe binary entries. Updates `bash-tools.exec-host-gateway.ts` and
`invoke-system-run.ts` to persist inner commands extracted from shell wrappers, ensuring
approval decisions are retained across sessions.

#### C0/C1 Control Character Escape ([`4508b818a`](https://github.com/openclaw/openclaw/commit/4508b818a))

Refactors `escapeInlineControlChars()` in `src/acp/event-mapper.ts` to use a
table-driven `INLINE_CONTROL_ESCAPE_MAP` constant instead of a switch statement.
Extends coverage to include all C0/C1 control characters in ACP resource link metadata,
preventing injection via control characters in resource links.

#### Tool-Call Name Validation ([`cdfe45eeb`](https://github.com/openclaw/openclaw/commit/cdfe45eeb))

Introduces a new `src/agents/pi-embedded-runner/tool-name-allowlist.ts` module with
`collectAllowedToolNames()` that builds a set of valid tool names from agent tools and
client tool definitions. Validation is enforced at multiple points: `compact.ts`,
`google.ts`, `attempt.ts`, `session-tool-result-guard-wrapper.ts`, and
`session-tool-result-guard.ts` — preventing persisted sessions from containing
fabricated tool-call names that could bypass tool policies.
