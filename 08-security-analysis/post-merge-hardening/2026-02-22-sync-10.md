## Hook Rate-Limit Unification ([`9f97555b5`](https://github.com/openclaw/openclaw/commit/9f97555b5))

Hook authentication rate limiting unified using `createAuthRateLimiter()` with scope `AUTH_RATE_LIMIT_SCOPE_HOOK_AUTH`. Replaces custom auth failure tracking with standardized patterns. Added configurable `pruneIntervalMs` to prevent memory leaks in long-running processes.

**Files:** `src/gateway/auth-rate-limit.ts`, `src/gateway/hooks-mapping.ts`, `src/gateway/server-http.ts`, `src/hooks/loader.ts`, `src/hooks/module-loader.ts` (NEW), `src/hooks/module-loader.test.ts` (NEW)

## Gateway Command/Audit Guardrails ([`265da4dd2`](https://github.com/openclaw/openclaw/commit/265da4dd2))

Two security improvements:

1. **Dangerous command detection:** New `collectNodeDangerousAllowCommandFindings()` in `src/security/audit-extra.sync.ts` audits `gateway.nodes.allowCommands` config for potentially dangerous commands (camera, screen, contacts, calendar, reminders, SMS).

2. **Rate limiting robustness:** `resolveControlPlaneRateLimitKey()` in `src/gateway/control-plane-rate-limit.ts` improved fallback for cases where device and client IP are unknown.

**Files:** CHANGELOG.md, docs/cli/security.md, docs/gateway/security/index.md, src/config/schema.help.ts, src/gateway/control-plane-rate-limit.ts, src/security/audit-extra.sync.ts, src/security/audit-extra.ts, src/security/audit.test.ts, src/security/audit.ts

## Avatar Symlink Escape Block ([`3d0337504`](https://github.com/openclaw/openclaw/commit/3d0337504))

Avatar path resolution in `src/gateway/session-utils.ts` now blocks symlink escapes. Validates that resolved avatar paths stay within expected directories, preventing path traversal via symlink chains.
